# final_project_python
Python_Final_Project


1. Описание проекта
Мы делаем простого Telegram‑бота для изучения слов иностранного языка.
Бот не требует отдельного сайта: всё общение идёт прямо в Telegram. Пользователь добавляет слова или берёт готовый набор, бот напоминает о повторении, задаёт вопросы по карточкам и ведёт простую статистику прогресса.
2. Как устроен и работает сервис
2.1. Архитектура
Вся логика живёт в одном монолитном приложении на Python:
Telegram‑бот на aiogram;
модуль работы с базой данных (PostgreSQL или SQLite);
модуль бизнес‑логики (словарь, тренировки, прогресс);
модуль с фоновой задачей, который рассылает напоминания.
Приложение и база поднимаются через docker-compose (один контейнер с приложением и один с БД).
2.2. Поведение для пользователя
Первый запуск
Бот здоровается, сохраняет user_id.
Спрашивает язык обучения, примерный уровень (A1–C1) и как часто напоминать о повторении слов.
Добавление слов
Можно взять готовый небольшой набор базовых слов.
Можно добавлять свои слова в формате «слово — перевод».
Тренировка
По команде «Начать тренировку» бот показывает карточки.
Пользователь отвечает (вариантами или текстом), бот даёт правильный ответ и фиксирует результат.
Напоминания
Раз в заданный интервал фоновой процесс ищет слова, которые пора повторить.
Бот пишет пользователю и предлагает пройти короткую сессию.
Прогресс
Бот хранит количество выученных слов, число тренировок и серию активных дней.
По команде «Мой прогресс» показывает эту статистику.
3. Основной функционал (MVP)
регистрация пользователя и его настроек;
хранение слов и простого прогресса по каждому слову;
режим тренировки с карточками;
напоминания о повторении через фоновые задачи;
просмотр базовой статистики.
4. Распределение ролей
Аркадий — NLP / ML
Продумывает и реализует логику повторения слов: как часто показывать слово в зависимости от ответов пользователя.
Готовит идеи по более продвинутым алгоритмам и возможному дальнейшему усложнению системы.
Ксения — Бизнес‑логика и база данных
Проектирует структуру БД: пользователи, слова, настройки, прогресс.
Пишет сервисы для добавления слов, сохранения результатов тренировок и подсчёта статистики.
Андрей — Сценарии и тексты внутри бота
Придумывает сценарии общения: приветствие, меню, тексты команд и подсказок.
Описывает форму тренировок: как выглядят вопросы, варианты ответов и сообщения после них.
Егор — Telegram‑бот на aiogram
Реализует бота: команды, кнопки, переходы между состояниями.
Связывает интерфейс с логикой Ксении и алгоритмами Аркадия.
Проверяет, что все основные сценарии для пользователя работают без сбоев.
Даня — Инфраструктура, фон и документация
Настраивает проект, Docker и docker-compose.
Делает фоновый планировщик, который рассылает напоминания.
Описывает, как запустить сервис, и рисует простую схему для отчёта.
5. Дополнительные фичи (на будущее)
Эти пункты не входят в базовый костяк, но их можно отдельно описать как планы развития:

диалоги‑персонажи и разбор письменных ответов с помощью языковых моделей;
